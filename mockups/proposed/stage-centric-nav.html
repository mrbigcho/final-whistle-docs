스테이지 중심 정보 구조
대진표·순위표·경기 탭을 대회 레벨에서 스테이지 레벨로 내려 포맷별 분기 문제를 해결한다. 문서: final-whistle/docs/대회_네비게이션_재편_제안.md
현재 구조 as-is
/competitions/:id
├─ 대진표 Elim에서만 유의미
├─ 경기 스테이지 칩으로 내부 분기
├─ 참가자
├─ 순위표 RR/Group에서만 유의미
└─ 정보
문제
· 포맷에 맞지 않는 탭이 항상 보임
· 매 탭마다 스테이지를 다시 선택
· 공유 URL이 스테이지를 특정 못함
제안 구조 to-be
/competitions/:id
├─ 참가자
├─ 스테이지 목록 → 각 스테이지 진입
│ └─ /stages/:stageId
│ ├─ 대진표 or 순위표 포맷별 자동
│ └─ 경기
└─ 정보
개선
· 대회 탭 5 → 3으로 축소
· 스테이지가 1급 개념으로 승격
· URL에 :stageId 포함 → 정확한 공유 링크

포맷별 서브탭 자동 선택

SingleElim · DoubleElim 엘리미네이션 계열
🎯 대진표 ⚔️ 경기
RoundRobin · GroupStage · LeaguePhase 리그 계열
🏅 순위표 ⚔️ 경기

기존 암묵적 경로는 라우터 redirect 훅에서 "현재 진행 중" 스테이지를 찾아 새 경로로 자동 이동한다.

상세 목업
Step 1 🏆 대회 상세 (스테이지 목록) 참가자 · 스테이지 · 정보 3탭. 스테이지 카드 목록 기본 화면. Step 2 🎯 Elim 스테이지 (대진표) SingleElim 8강 진입. 첫 탭이 대진표, 순위표 탭은 미노출. Step 3 🏅 RR 스테이지 (순위표) GroupStage 진입. 첫 탭이 순위표, 대진표 탭은 미노출. 조별 칩. Step 4 ⚔️ 경기 탭 (공통) 모든 포맷의 두 번째 서브탭. 라운드별 그룹 + 필터 칩.

단일 스테이지 대회 UX — 대회의 스테이지가 1개뿐이면 "스테이지 목록" 화면을 건너뛰고 해당 스테이지 상세로 바로 진입한다. "스테이지" 탭은 실질적으로 해당 스테이지의 대진표/순위표 화면과 동일해진다.
